## 4. 대시보드 화면 구성

```
┌────────────────────────────────────────────────────────┐
│  헤더: 로고 / 스캔 상태 배지 / 현재 시각               │
├──────────────┬─────────────────────────────────────────┤
│              │  탭 바: 모니터링 | 결과 | 보고서         │
│  사이드바    ├─────────────────────────────────────────┤
│  ─────────   │  [모니터링 탭]                           │
│  스캔 설정   │  ┌────────────┐ ┌────────────────────┐  │
│  모듈 목록   │  │ 파이프라인 │ │ 스캔 진행 상태 바  │  │
│  요약 현황   │  │ (모듈 순서)│ │ 파일 분석 진행 바  │  │
│  마지막 스캔 │  │            │ │ 실시간 로그        │  │
│              │  └────────────┘ └────────────────────┘  │
└──────────────┴─────────────────────────────────────────┘
```

### 사이드바 주요 기능

- **스캔 대상 IP** — 점검할 대상 IP 주소 입력 (기본: 127.0.0.1)
- **개인정보 스캔 경로** — PII 탐지 대상 폴더 지정
- **보고서 형식** — HTML / JSON 선택
- **모듈 토글** — 각 모듈 ON/OFF 개별 설정
- **스캔 시작 / 중단** 버튼

## 5. 스캔 모듈 설명

| # | 모듈 | 기본값 | 주요 점검 내용 |
|---|------|--------|----------------|
| 1 | 서버 상태 검사 | ON | CPU·메모리·디스크 사용률, 방화벽·UAC·Windows Defender 상태 |
| 2 | 네트워크 트래픽 분석 | ON | 활성 연결 목록, 위험 포트(445·3389·23 등), 외부 IP 통신 |
| 3 | 프로세스 분석 | ON | 악성 프로세스 키워드, PPID 스푸핑, 숨겨진 프로세스 탐지 |
| 4 | 로그 분석 | ON | Windows 이벤트 로그, SQLi·XSS 공격 패턴, 로그인 실패 급증 |
| 5 | 외부 공격 탐지 | ON | RDP·SMB 브루트포스, 포트 스캔 흔적, 비정상 트래픽 |
| 6 | 포트 스캔 | ON | 1040개 포트 병렬 TCP 스캔, 열린 포트 위험도 분류 |
| 7 | 개인정보 유출 탐지 | **OFF** | 주민번호·카드번호·휴대폰·이메일·API키 파일 내 탐지 |
| 8 | 악성코드 탐지 | ON | Windows Defender 연동, PUP/애드웨어, 랜섬웨어 지표, RAT |

> **개인정보 탐지 모듈**은 기본값이 OFF입니다. 사이드바에서 토글을 켜고 스캔 경로를 지정하세요.

## 6. 스캔 실행 방법

1. 브라우저에서 `http://localhost:5001` 접속
2. 사이드바에서 **스캔 대상 IP** 확인 (로컬 점검 시 `127.0.0.1`)
3. 실행할 모듈을 토글로 선택
4. **[스캔 시작]** 버튼 클릭
5. 모니터링 탭에서 진행 상태 확인
   - **스캔 진행 바** — 현재 모듈 번호와 이름
   - **파일 분석 진행 바** — 파일 스캔 시 개별 파일 진행률
   - **실시간 로그** — 각 단계별 상세 로그
6. 스캔 완료 시 자동으로 **결과 탭**으로 이동

### 스캔 중단

- **[중단]** 버튼 클릭 → 진행 중인 모듈 완료 후 중단
- 스캔이 멈춘 것처럼 보일 경우 **[강제 초기화]** 버튼 사용

## 7. 결과 해석

### 심각도 분류

| 심각도 | 색상 | 의미 | 즉시 조치 |
|--------|------|------|-----------|
| 심각 (Critical) | 🔴 빨강 | 즉각적인 위협, 악성코드·랜섬웨어 등 | **즉시** |
| 높음 (High) | 🟠 주황 | 높은 위험, 취약한 설정·의심 프로세스 | 당일 |
| 중간 (Medium) | 🟡 노랑 | 보안 개선 필요 | 1주일 내 |
| 낮음 (Low) | 🔵 파랑 | 참고 사항, 모니터링 권고 | 1개월 내 |
| 정보 (Info) | ⚪ 회색 | 현황 정보 | 조치 불필요 |

### 결과 카드 구성

각 탐지 항목(Finding)은 다음 정보를 포함합니다:

- **제목** — 탐지 내용 요약
- **설명** — 구체적인 위험 내용
- **상세** — 영향 파일 목록, 탐지 패턴, 수치
- **폴더 열기** — 해당 파일/경로를 탐색기에서 바로 열기
- **기술 권고 및 조치 명령** — PowerShell/CMD 명령어 포함 조치 방법

### 결과 필터링

- 상단 심각도 카드 클릭 → 해당 심각도만 표시
- 모듈별 필터 버튼으로 특정 영역 결과만 확인

## 8. 개인정보(PII) 스캔 설정

사이드바 **개인정보 스캔 경로** 입력란에 탐색할 폴더를 입력합니다.

| 예시 입력 | 설명 |
|-----------|------|
| (빈 값) | 문서·바탕화면·다운로드 폴더 자동 탐색 |
| `C:\` | C드라이브 전체 (시간 소요 큼) |
| `C:\Users\사용자이름\Documents` | 특정 폴더만 |

### 탐지 패턴

| 유형 | 예시 | 심각도 |
|------|------|--------|
| 주민등록번호 | `880101-1234567` | Critical |
| 신용카드 번호 | `1234-5678-9012-3456` | Critical |
| 비밀번호 평문 | `password=mysecret` | High |
| API 키 | `api_key=sk-xxxx` | High |
| 휴대폰 번호 | `010-1234-5678` | Medium |
| 이메일 주소 | `user@example.com` | Low |
| 계좌번호 | `123-456789-01-234` | High |
| 내부 IP 주소 | `192.168.x.x` | Low |

> 탐지된 결과의 **오탐 주의** 항목을 반드시 확인하세요.  
> Python·npm 패키지 내 파일은 정상일 수 있습니다.

## 9. 보고서 저장

스캔 완료 후 자동으로 보고서가 저장됩니다.

| 형식 | 파일명 | 내용 |
|------|--------|------|
| HTML | `security_report.html` | 브라우저에서 바로 열 수 있는 보고서 |
| JSON | `security_report.json` | 자동화 분석용 원시 데이터 |

저장 위치: 프로그램 실행 폴더 (`security_scanner/`)

## 10. 주의 사항

- 반드시 **관리자 권한**으로 실행하세요. 권한 부족 시 일부 모듈 결과가 누락됩니다.
- 포트 스캔 모듈은 **자기 자신(localhost)** 점검 용도입니다. 타인의 시스템에 허가 없이 사용하지 마세요.
- 개인정보 탐지 모듈은 C드라이브 전체 스캔 시 수 분~수십 분이 소요될 수 있습니다.
- 랜섬웨어 감염 지표로 탐지된 항목 중 Python·npm 패키지 내 `README.txt`, 백업 소프트웨어의 `RecoveryFiles.ini` 등은 정상 파일일 수 있습니다.
- Windows Defender가 비활성화된 환경에서는 악성코드 탐지 일부 기능이 제한됩니다.
